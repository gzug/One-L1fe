# Android Health Connect — Native Stub Contract

> This file documents the *intended* native configuration generated by `expo prebuild`.
> Do **not** manually commit `android/` files — they are overwritten on every prebuild.

## What `expo prebuild` generates

### `android/app/src/main/AndroidManifest.xml` (relevant additions)

```xml
<!-- Health Connect data type permissions -->
<uses-permission android:name="android.permission.health.READ_STEPS" />
<uses-permission android:name="android.permission.health.READ_HEART_RATE" />
<uses-permission android:name="android.permission.health.READ_ACTIVE_CALORIES_BURNED" />
<uses-permission android:name="android.permission.health.READ_DISTANCE" />
<uses-permission android:name="android.permission.health.READ_SLEEP" />

<!-- Required for Health Connect privacy policy intent -->
<activity
  android:name="androidx.health.connect.client.permission.HealthPermissionActivity"
  android:exported="true" />
```

### `android/app/src/main/java/…/MainActivity.kt` (relevant additions)

```kotlin
// react-native-health-connect registers a ReactActivityDelegate override.
// No manual edits needed — the plugin handles this via AppDelegate pattern.
class MainActivity : ReactActivity() {
  // PLACEHOLDER: plugin injects onRequestPermissionsResult override here
  // to route Health Connect permission callbacks back to JS bridge.
}
```

## Controlled via `app.json`

```json
{
  "expo": {
    "plugins": [
      [
        "react-native-health-connect",
        {
          "permissions": [
            "Steps",
            "HeartRate",
            "ActiveCaloriesBurned",
            "Distance",
            "SleepSession"
          ]
        }
      ]
    ]
  }
}
```

## Verification

After running `cd apps/mobile && npx expo prebuild --platform android --clean`:

1. Confirm `android/app/src/main/AndroidManifest.xml` contains the five `READ_*` permissions above.
2. Confirm `android/app/src/main/java/…/MainActivity.kt` does **not** override `onRequestPermissionsResult` manually (the plugin handles it).
3. Run `wearablePermissions.assertions.ts` (no device needed) to verify the JS-layer permission flow.
4. For real-device validation see `WEARABLE-TD-001` in `docs/garmin-data-path.md`.
